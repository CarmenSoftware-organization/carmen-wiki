# โปรไฟล์ (Profile)

> **At a Glance**
> **วัตถุประสงค์ของโมดูล:** หน้าจัดการตนเองสำหรับผู้ใช้ที่ล็อกอินอยู่ใช้ดูและแก้ไขข้อมูลตัวตนของตน รวมถึงเปลี่ยนรหัสผ่าน &nbsp;·&nbsp; **กลุ่มผู้ใช้:** ผู้ใช้ที่ล็อกอินอยู่ (เจ้าของบัญชีเอง) &nbsp;·&nbsp; **เอนทิตี/ตารางหลัก:** `user`, `user_info`, `business_unit` (แสดงแบบอ่านอย่างเดียว) &nbsp;·&nbsp; **หน้าย่อย:** 0

## 1. ภาพรวม

Profile คือหน้าบัญชีส่วนตัวของผลิตภัณฑ์ Carmen Platform admin ผู้ใช้ที่ผ่านการพิสูจน์ตัวตนทุกคนจะเข้าถึงหน้านี้ได้จากเมนู avatar ที่ด้านล่างของแถบเมนูข้าง — ไม่มีรายการนำทางระดับบนสำหรับ `/profile` หน้านี้แสดงข้อมูลตัวตนปัจจุบันของผู้ใช้ (alias name, ชื่อจริง/ชื่อกลาง/นามสกุล, เบอร์โทรศัพท์, อีเมล, platform role, รหัสบัญชี, วันที่เป็นสมาชิก) และในการ์ดแบบอ่านอย่างเดียวอีกใบหนึ่ง แสดงรายการ business unit ที่ถูกกำหนดให้บัญชีนั้น

รองรับการเขียนข้อมูลสองรูปแบบ แบบแรกคือการแก้ไขฟิลด์ตัวตน — alias name, ชื่อจริง, ชื่อกลาง, นามสกุล, เบอร์โทรศัพท์ — ผ่าน `PUT /api/user/profile` โดยอีเมลไม่สามารถแก้ไขจากหน้านี้ได้ แบบที่สองคือการเปลี่ยนรหัสผ่านบัญชีผ่าน modal dialog ที่ต้องกรอกรหัสผ่านปัจจุบันพร้อมรหัสผ่านใหม่ความยาวอย่างน้อยหกตัวอักษร และก็ส่งไปยัง `PUT /api/user/profile` เช่นกัน การบันทึก identity edit จะ re-fetch ข้อมูลโปรไฟล์และ refresh local auth context ทำให้ avatar และชื่อที่แสดงในแถบเมนูข้างอัปเดตทันที การเปลี่ยน password ไม่ได้ refresh auth context — form เพียงแค่ปิดลง

หน้านี้มีขอบเขตที่แคบโดยตั้งใจ ไม่กำหนดหรือยกเลิกการเป็นสมาชิก business unit ไม่เปลี่ยน platform role และไม่จัดการผู้ใช้คนอื่น — flow เหล่านั้นเป็นของโมดูล [[users]] และ [[auth-roles]] ซึ่งต้องการสิทธิ์ admin Profile เป็นหน้า Platform เพียงหน้าเดียวที่บัญชีที่ผ่านการพิสูจน์ตัวตนทุกบัญชีเข้าถึงได้โดยไม่คำนึงถึง role

## 2. บริบททางธุรกิจ

Profile เป็นหน้าจัดการตนเองแบบ self-service ไม่มีปัจจัยทางธุรกิจภายนอกที่ขับเคลื่อนนอกเหนือจากการรักษาให้ข้อมูลตัวตนของผู้ใช้แต่ละคนเป็นปัจจุบัน เพื่อให้ audit log การแจ้งเตือน และรายชื่อสมาชิก BU อ้างอิงชื่อและข้อมูลติดต่อที่ถูกต้อง

## 3. แนวคิดสำคัญ

- **Profile**: ชุดของฟิลด์ตัวตนที่เป็นของบัญชีผู้ใช้แต่ละคน — alias name, ชื่อจริง/ชื่อกลาง/นามสกุล, เบอร์โทรศัพท์, อีเมล ทั้งหมดแก้ไขได้จากหน้านี้ยกเว้นอีเมล
- **Alias name**: ป้ายชื่อสั้น ๆ ที่ใส่ได้หรือไม่ใส่ก็ได้ ใช้สำหรับแสดงตัวอักษรย่อบน avatar และเป็นชื่อย่อในที่ที่พื้นที่จำกัด
- **Email (เปลี่ยนไม่ได้)**: รหัสประจำตัวสำหรับล็อกอินของผู้ใช้ แสดงในหน้า Profile แบบอ่านอย่างเดียว การเปลี่ยนค่านี้เป็นการดำเนินการระดับ administrative ที่จัดการนอกโมดูลนี้
- **การเปลี่ยนรหัสผ่าน**: flow แบบ modal เฉพาะที่ต้องการรหัสผ่านปัจจุบัน รหัสผ่านใหม่ (อย่างน้อยหกตัวอักษร) และค่ายืนยันที่ตรงกัน ส่งผ่าน endpoint `PUT /api/user/profile` เดียวกันกับการแก้ไขข้อมูลตัวตน แต่ใส่ฟิลด์รหัสผ่านแทน
- **Business unit ที่ถูกกำหนด**: รายการ BU ที่ผู้ใช้สังกัด แสดงเป็นการ์ดแบบอ่านอย่างเดียว การเป็นสมาชิกจัดการในโมดูล [[users]] โดยผู้ดูแลระบบ หน้า Profile เพียงแสดงข้อมูลเท่านั้น
- **รหัสบัญชีและวันที่เป็นสมาชิก**: ข้อมูล metadata แบบอ่านอย่างเดียวที่ประทับเวลาตอนสร้างบัญชี มีประโยชน์สำหรับการสนทนากับ support และ audit แต่แก้ไขจากหน้านี้ไม่ได้

## 4. บทบาทและ Persona

ใช้โดยผู้ใช้ที่ล็อกอินอยู่ (เจ้าของบัญชีเอง) ไม่มี role admin ใดที่เกี่ยวข้อง

## 5. โมดูลที่เกี่ยวข้อง

- [[users]] — โมดูลฝั่ง administrative ที่สร้างบัญชี กำหนด business unit และตั้ง platform role; Profile เพียงอ่านสิ่งที่ Users เขียนเท่านั้น
- [[auth-roles]] — เป็นเจ้าของ flow การล็อกอินและการกำหนด role ที่ควบคุมว่าผู้ใช้จะเข้าถึง Profile ได้หรือไม่ตั้งแต่แรก
- [[business-units]] — แหล่งของรายการ BU ที่แสดงแบบอ่านอย่างเดียวบนหน้า Profile
